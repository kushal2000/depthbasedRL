IsaacGym → Isaac Lab Migration Checklist
=========================================

This documents every change made during the migration so you can verify
that the Isaac Lab training matches the IsaacGym training.

1. PHYSICS PARAMETERS
---------------------
IsaacGym                          →  Isaac Lab
dt = 1/60, substeps = 2          →  dt = 1/120, decimation = 2
  effective physics dt: 1/120        effective physics dt: 1/120
  control rate: 1/60                 control rate: 1/60
episodeLength = 600 (steps)       →  episode_length_s = 10.0 (seconds)
  600 * (1/60) = 10.0s              10.0 / (1/60) = 600 steps
solver_type = 1 (TGS)            →  solver_type = 1 (TGS)
num_position_iterations = 8       →  max_position_iteration_count = 8
num_velocity_iterations = 0       →  max_velocity_iteration_count = 0
bounce_threshold_velocity = 0.2   →  bounce_threshold_velocity = 0.2
max_depenetration_velocity = 1000 →  Set on RigidBodyPropertiesCfg (not PhysxCfg)
angular_damping = 0.01            →  Set on RigidBodyPropertiesCfg
linear_damping = 0.01             →  Set on RigidBodyPropertiesCfg

CHECK: Verify effective physics rate is identical (120 Hz physics, 60 Hz control).

2. QUATERNION CONVENTION
------------------------
IsaacGym: xyzw (x, y, z, w)
Isaac Lab: wxyz (w, x, y, z)

All quaternion operations in the env code need conversion at API boundaries:
  - robot.data.body_quat_w returns wxyz
  - object.data.root_quat_w returns wxyz
  - write_root_pose_to_sim expects wxyz
  - Isaac Lab math functions (quat_mul, quat_rotate) expect wxyz

The env internally keeps xyzw for reward/obs math compatibility and converts
at Isaac Lab API boundaries using quat_xyzw_to_wxyz() / quat_wxyz_to_xyzw().

CHECK: Log object_rot and palm_rot from both envs and compare.

3. JOINT ORDERING
-----------------
IsaacGym: depth-first traversal of kinematic tree
Isaac Lab: breadth-first traversal of kinematic tree

The env builds runtime index maps via _build_joint_index_maps():
  _all_joint_ids: maps our canonical order → Isaac Lab joint indices
  _inv_joint_ids: maps Isaac Lab joint indices → our canonical order

Actions are applied using: robot.set_joint_position_target(targets[:, _all_joint_ids])
Observations read using:   joint_pos = robot.data.joint_pos[:, _all_joint_ids]

       The canonical order should be: iiwa14_joint_1..7, then left_thumb_*,
       left_index_*, left_middle_*, left_ring_*, left_pinky_*.

10. RESET LOGIC
----------------
IsaacGym order:
  post_physics_step() → compute_reward() → compute_observations() → reset_idx()
  Observations returned are PRE-reset (stale obs for reset envs).

Isaac Lab order:
  _get_dones() → _get_rewards() → _reset_idx() → _get_observations()
  Observations returned are POST-reset (fresh obs for reset envs).

CHECK: This is a known difference. The RL agent should handle it correctly
       since it uses the obs from the NEXT step anyway (value bootstrap).

11. REWARD COMPUTATION
-----------------------
All reward functions should produce identical values for identical states:
  - lifting_reward: distance above table, bonus for height > threshold
  - keypoint_reward: distance between object keypoints and goal keypoints
  - distance_delta: change in keypoint distance between steps
  - action_penalty: L2 penalty on arm and hand actions separately
  - reach_goal_bonus: bonus when all keypoints within tolerance
  - fall_penalty: penalty when object falls below threshold

CHECK: Log per-component rewards from both envs for the same scenario.
       The reward_shaper scale (0.01) is applied in the RL agent, not the env.

17. ISAAC LAB API GOTCHAS (discovered during testing)
------------------------------------------------------
a) max_episode_length is a READ-ONLY PROPERTY on DirectRLEnv.
   It's computed as: ceil(cfg.episode_length_s / (cfg.sim.dt * cfg.decimation))
   Do NOT set self.max_episode_length in __init__. Use self.max_episode_length directly.

b) step_dt is also a property. Use self.step_dt instead of self.dt.
   step_dt = cfg.sim.dt * cfg.decimation = 1/120 * 2 = 1/60

c) DirectRLEnvCfg.num_actions/num_observations/num_states are DEPRECATED.
   Use action_space/observation_space/state_space (int) directly on the cfg.

d) gymnasium.make() wraps env in OrderEnforcing wrapper.
   To access .device, .cfg, .num_envs use env.unwrapped.* instead.

e) TiledCamera depth convention:
   - IsaacGym: returns NEGATIVE depth → negate to get positive
   - Isaac Lab TiledCamera: returns POSITIVE depth → use as-is
   Both are then clamped to [0, depth_far] and normalized to [0, 1].

f) Table USD from URDF converter has an ArticulationRoot + root_joint.
   Must disable articulation: articulation_props=ArticulationRootPropertiesCfg(
       articulation_enabled=False)
   This produces a harmless PhysX error about "cannot create a joint between
   static bodies" but doesn't affect simulation.

g) Joint init state regex patterns cannot overlap.
   ".*" = 0.0 conflicts with "iiwa14_joint_1" = -1.571.
   Use "left_.*" = 0.0 instead of ".*" = 0.0.

h) ImplicitActuatorCfg.effort_limit is deprecated.
   Use effort_limit_sim instead (produces a warning but still works).

i) EventCfg for domain randomization:
   DirectRLEnvCfg has a built-in 'events' field consumed by EventManager.
   Custom DR params must use a DIFFERENT field name (e.g., 'dr_params')
   to avoid Isaac Lab's EventManager trying to parse them as EventTermCfg.

j) PhysxCfg does NOT have max_depenetration_velocity.
   That property is on RigidBodyPropertiesCfg instead.

k) ImplicitActuatorCfg is in isaaclab.actuators, NOT isaaclab.sim.

l) Python output buffering with Isaac Sim:
   When running via background process, stdout is fully buffered by default.
   Use PYTHONUNBUFFERED=1 or python -u to get real-time output.
   Without this, it can appear as if the process is hung when it's actually running.

m) SAPGAgent.train_epoch() requires init_tensors() first:
   init_tensors() is called by train(), not the constructor.
   For testing, call agent.init_tensors() + agent.obs = agent.env_reset()
   before calling train_epoch() directly.

n) Multi-GPU requires --distributed flag for AppLauncher:
   AppLauncher(distributed=True) reads LOCAL_RANK/RANK/WORLD_SIZE from torchrun.
   Each rank must set env_cfg.sim.device = f"cuda:{local_rank}".
   Use app_launcher.local_rank and app_launcher.global_rank (not os.getenv).
   Launch: torchrun --nproc_per_node=N train_isaaclab.py --distributed --multi_gpu

o) NCCL "Call to bind failed: Address already in use" warning is harmless:
   This is from NCCL's RAS monitoring socket, does not affect training.

p) Multi-GPU + depth camera LIMITATION:
   Isaac Sim's usdrt rendering only supports cuda:0:
     "GPU 1 requested. GPUs other than cuda:0 are not currently supported"
   This means TiledCamera depth rendering FAILS on any GPU other than 0.
   Multi-GPU training WITHOUT depth works fine (each rank on its own GPU).
   Multi-GPU WITH depth requires workaround:
     Option 1: All rendering on GPU 0, transfer depth tensors to other GPUs
     Option 2: Use separate Isaac Sim processes (each gets cuda:0 in its CUDA_VISIBLE_DEVICES)
     Option 3: Disable depth in multi-GPU, train with depth on single GPU only

21. PERFORMANCE BENCHMARKS (Isaac Lab vs IsaacGym)
----------------------------------------------------
2048 envs, single GPU (RTX 6000 Ada 48GB):

                    Isaac Lab       IsaacGym
No depth:           44,024 FPS      60,517 FPS
With depth:         10,272 FPS       2,413 FPS
Depth overhead:     ~4.3x           ~25x

Key finding: Depth rendering is 4.3x faster in Isaac Lab thanks to
TiledCamera batched rendering (vs per-env sequential in IsaacGym).
Without depth, IsaacGym is ~1.4x faster (lower overhead from simpler API).

256 envs with depth:  2,716 FPS (94ms/step)
4096 envs with depth: 13,421 FPS (305ms/step)

Note: 8192 envs takes >10min to create (USD stage setup is slow at scale).
      Env creation time: 16 envs ~8s, 256 ~13s, 2048 ~75s, 4096 ~184s.
